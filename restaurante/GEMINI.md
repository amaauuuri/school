#Project Context: Restaurant POS SaaS

##Team & Objective
* **Team Lead:** Amau (Information & Communication Technologies TICs Engineering student)


* **Team Members:** Cesar, Balanzar, Steph, and probably Zaira.
* **Commercial Goal:** Build and launch a fast, reliable MVP for local restaurants to bootstrap an international volunteering trip.

## Tech Stack
* **Frontend:** React + Vite (Single Page Application, clean UI).
* **Backend:** Node.js + Express (Clean REST API).
* **Database:** PostgreSQL (Relational database for financial data integrity and cash flow accuracy).
* **Development Strategy:** Vibe Coding + layered learning (MVP → Best Practice → Scalable).

## Architecture & Business Rules
1. **No complex physical hardware in V1:**After that we think in physical thermal printers or local networks for now. Everything is digital (kitchen/cashier screens, digital tickets sent via Web/WhatsApp).
2. **Business Model:** Cloud SaaS (Monthly/Annual subscription).
3. **Financial Security:** Table status, sales, orders, and logs must be 100% consistent (Strict relational database constraints).
4. **Code Quality:** Standard JavaScript/TypeScript. Avoid deprecated packages or unsafe shortcuts
5. **Language Constraint:** Always respond in English. Keep code, comments, and technical explanations professional and concise.
